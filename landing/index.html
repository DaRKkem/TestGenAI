What it does

Structured code understanding

For Python, TestGen AI parses your code with the ast module to extract real function signatures. For every other supported language, it uses prompt chaining — one pass analyzes the code, a second writes the tests.

Never blocked by one provider

Every generation runs a health check first. If Mistral AI is unavailable, TestGen AI automatically retries on Groq's LPU-backed infrastructure — so a single provider outage never stops your work.

Every generation, kept

Revisit past runs at any time: source code, generated tests, and provider used are all saved to your history, with one click to reopen a run straight back into the editor.

About the project

Writing unit tests is one of those tasks every developer knows they should do more of — normal cases, edge cases, error cases — and one of the first things that gets rushed when a deadline is close. I kept hitting that same wall on my own projects, and at the same time I wanted to build something with large language models that went further than a chatbot: a tool that actually reads code and hands back something a developer can drop straight into their test suite.

TestGen AI is where those two ideas met. It's my end-of-first-year Portfolio Project at Holberton School, designed, built, and shipped solo over 11 weeks — backend, AI integration, frontend, and documentation all my own.
